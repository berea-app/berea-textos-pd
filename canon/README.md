# canon/

Canonical book identifiers, in lockstep with the Berea Android app.

- [`canon_66.json`](canon_66.json) — protestant canon (39 OT + 27 NT). Each
  entry: `book_id` (USFM 3.0 lowercase, frozen), `order`, `testament`,
  `section`, `name_es`, `name_en`, `abbr_es`, `chapters` (RV 1909 chapter
  count), `aliases`, optional `numbering_note`.
- [`canon_extendido.json`](canon_extendido.json) — deuterocanonical and
  LXX-only books distributed by Berea (`tob`, `jdt`, `wis`, `sir`, `bar`,
  `lje`, `s3y`, `sus`, `bel`, `1ma`, `2ma`, `3ma`, `4ma`, `man`, `1es`, `2es`,
  `oda`, `pss`, `ps2`, plus the embedded `esg`). Each entry adds
  `parent_book_id` (when applicable) and `canon_families[]`.
- [`verse_alias_lxx.json`](verse_alias_lxx.json) — generated by the pipeline
  to map between LXX and MT numbering of the Psalms (and a few other places
  with divergent numbering). The Berea app loads this into `verse_alias`
  in SQLite so that cross-translation references work without ambiguity.

These files are the **single source of truth** shared between the public-domain
text repository and the Android app. They are intentionally duplicated in
both repositories because each one needs to ship them as an asset; the
duplication is enforced by CI.

## Why USFM 3.0 lowercase

USFM 3.0 is what Paratext emits, OSHB consumes, and STEPBible exposes. Using
the lowercase identifiers keeps the canon compatible with the entire ecosystem
of academic Bible tooling without any lookup tables.

## Adding new books

If a future release of Berea ships books beyond the current 66+20, add them
here first, get sign-off in a PR, and only then update the parsers. Never let
the parser invent a `book_id`.
